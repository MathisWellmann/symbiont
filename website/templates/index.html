LLMs write type-safe Rust function bodies that get compiled natively and hot-swapped into your running binary — bare-metal execution, zero interpreter overhead.

How it works

LLMwrites Rust fn
→
Validateparse + signature
→
Compilenative .so
→
Hot-swapinto binary
→
Evaluatebare-metal
↺

Constrained generation makes this reliable: parse errors, signature mismatches, and compiler diagnostics are fed back to the LLM automatically until it produces valid, compilable code.

Quick Start

symbiont::evolvable! {
    fn step(counter: &mut usize) {
        *counter += 1;  // default impl, evolved by the LLM
    }
}

#[tokio::main]
async fn main() -> symbiont::Result<()> {
    let runtime = symbiont::Runtime::init(SYMBIONT_DECLS, Profile::Debug).await?;
    let agent = symbiont::inference::init_agent()?;

    loop {
        step(&mut counter);  // bare-metal: native dylib call

        // LLM rewrites the fn, harness validates + compiles + hot-swaps
        runtime.evolve_with_backpressure(&agent, &prompt).await?;
    }
}

Core Highlights

Type-safe agentic code

Agents express intent as Rust functions with enforced signatures. The compiler is the guardrail.

Constrained generation

Parse errors, signature mismatches, and compiler diagnostics steer the LLM until it produces valid code.

Hot-swap dylibs

Functions compile to native shared libraries and swap in-place via libloading — no process restart.

Bare-metal performance

~1 ns dispatch overhead. Lock-free hot path via AtomicPtr. Multi-thread safe.

Panic catching

Runtime panics in LLM code are caught inside the dylib and fed back as prompt context automatically.

Plug-in inference

Any OpenAI-compatible provider via rig. Local or cloud.
